Данный гайд создал для PostgreSQL на Astra Linux (ядро Debian)

1. Создаем sh файл, где указываем:

REPO_PATH = путь до PostgreSQL на компе
BACKUP_FOLDER = куда сохранять

Создаем дамп нашей БД и при помощи gzip архивируем его, > направляет созданое творение по указанному пути с именем MY_DB-ДАТА.sql.gz 
pg\_dump MY_DB | gzip > $BACKUP_FOLDER/MY_DB-$(date +%Y-%m-%d).sql.gz 

2. Делаем файл исполняемым, в bash прописываем 
chmod +x имяфайла.sh

Если вам не нужна автоматика, а хотите запускать ручками, то тут можно и закончить

3. cron

Чтобы посмотреть, какие есть задачи вообще прописываем в bash
crontab -l

Чтобы добавить задачу в cron прописываем:
sudo crontab -e
Он предложит выбрать редактор, я например использую nano, но это вкусовщина, можете хоть кровью по экрану мазать, лишь бы работало

Мы увидим манускрит из сплошных комментов, там объясняется что вообще происходит и как добавить задачу. Полезно почитать.
Мы же пишем (без решетки/комента) следующее: 
55 16 * * * bash /backups/PostgreSQL/test-backup.sh
Где цифры/звездочки это переодичность
55 - минуты, 16 - час, *,*,* - любой день месяца, месяц, день недели

А дальше указываем путь до исполняемого файла, в моем случае это test-backup.sh
